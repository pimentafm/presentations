O problema

Por que uma IDE?

Dado geoespacial vive em silos, formatos divergentes e projeções incompatíveis. Uma IDE é o catálogo + gateway + contrato que torna esse caos operável.

01 · SILOS
Cópias divergentes
N times mantêm versões diferentes do mesmo shapefile — sem ground truth.
02 · DISCOVERY
Sem descoberta

03 · INTEROP
Babel de formatos
KML, SHP, GeoJSON, GDB, GeoPackage — cada cliente fala um dialeto.
04 · VENDOR
Lock-in
Esri SDE, Oracle Spatial, Mapbox Tiles — sair custa caro e quebra clientes.

SHEET 03 / 19
Vector & raster

Tipos de dado espacial.

Todo dado geo se reduz a duas formas: vector (geometria discreta) ou raster (matriz contínua) — mais 3 variantes especializadas.

Geometria discreta

Vector

Ponto, linha, polígono — coordenadas + atributos.

  • Ponto · POIs, sensores, eventos
  • Linha · rotas, rios, malha viária
  • Polígono · limites, lotes, biomas
  • Formatos · GeoJSON, Shapefile, GeoPackage, PostGIS
Matriz contínua

Raster

Grades de pixels georreferenciadas + bandas.

  • Imagery · Landsat, Sentinel, drone
  • Elevação · DEM, DTM, batimetria
  • Categorical · uso do solo, cobertura
  • Formatos · GeoTIFF, COG, Zarr
3D · point clouds (LiDAR) · meshes · 3D Tiles 4D · time-dynamic · telemetria · NDVI Tiles · pirâmides pré-renderizadas (XYZ, WMTS)
Vector + raster + variantes CADA TIPO → ÍNDICE PRÓPRIO · SERVIÇO PRÓPRIO

A fundação

PostGIS

O mesmo PostgreSQL que você já monitora e escala. PostGIS estende o catálogo com geometria, raster e ~500 funções — sem runtime externo, sem porta extra. A operação permanece idêntica.

  • HA familiar — Pgpool-II + PgBouncer + streaming replication funcionam normalmente.
  • Backup padrão — pg_dump, pgBackRest, WAL archiving — sem ferramental geo dedicado.
  • Observability padrão — pg_stat_statements, pg_stat_io, postgres_exporter capturam queries espaciais como qualquer outra.
  • Atenção — índices GiST/SP-GiST custam mais que B-tree. Plano de execução muda; instrumentar antes de prod.
-- Toda área protegida num raio de 5km do ponto SELECT nome, ST_Area(geom::geography) AS area_m2 FROM areas_protegidas WHERE ST_Intersects(geom, ST_Buffer(:ponto, 5000));

SHEET 08 / 19
Reescrita cloud-native

GeoServer Cloud

Mesma feature parity OGC. Repensado para Kubernetes — cada serviço autônomo, estado externalizado, scaling independente.

  • Microsserviços por OWS — wms-service, wfs-service, wcs-service, rest-service, web-ui, gwc. Cada um escala isolado.
  • Stateless workers — configuração externalizada via Spring Cloud Config Server. Pods são fungíveis.
  • Service discovery — Consul (ou profile standalone que usa discovery nativo do K8s).
  • Event bus — RabbitMQ propaga mudanças de catálogo em hot, sem reinício de pod.

Catálogo & governança

Sem catálogo, dado é arquivo solto. Duas camadas complementares: ISO-19115 para descritivo, OpenMetadata para operacional.

Service registry geo

Catálogo CSW + editor ISO. O service-discovery do mundo geo.

  • ISO 19115-3 / 19119 / 19110 + CSW 2.0.2
  • Harvesting: CSW, WFS, OAI-PMH, SFTP
  • Editor multilíngue + workflow de publicação
  • Federação nativa entre instâncias
Data catalog corporativo

OpenMetadata

Lineage + qualidade + governança no mesmo plano de dados não-geo.

  • Lineage cross-pipeline (camada → tabela → S3)
  • Data quality framework + alertas de drift
  • 120+ connectors (bancos, BI, ML, geo)
  • Glossário, ownership, RACI

SHEET 14 / 19
A ponte entre LLMs e serviços geo

MCP Server Geospatial

Model Context Protocol expõe ferramentas geo como tools tipadas para AI agents — os mesmos serviços OGC agora servem agents, não só browsers.

TOOL · QUERY
PostGIS Query
Agent executa ST_Intersects, ST_Buffer, ST_Within via SQL parametrizado. Resultado tipado em GeoJSON.
TOOL · PUBLISH
GeoServer Layers
Criar, atualizar e remover camadas WMS/WFS programaticamente. Style em SLD gerado pelo LLM.
TOOL · RENDER
Tile Rendering
Renderizar tiles sob demanda, seed cache GWC, gerar previews de camadas para validação visual.
TOOL · DISCOVER
Catalog Search
Buscar metadados ISO 19115-3 via CSW. Agent descobre datasets por tema, BBOX ou temporal extent.
TOOL · ANALYZE
Raster Analysis
NDVI, classificação, álgebra de mapas. Agent processa COGs e retorna estatísticas zonais.
TOOL · GEOCODE
Geocoding
Forward/reverse geocoding. Resolver endereços em coordenadas e vice-versa para contexto espacial.

Monitoramento de desmatamento

Decisão autônoma

Agent loop

  • Avalia qualidade da imagery (cloud cover < 20%)
  • Seleciona modelo por tipo de análise
  • Valida resultado contra ground truth anterior
